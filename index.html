Open vs Closed Weights Concept

Weights are the learned parameters of a neural network — billions of numbers that encode "knowledge." Whether they're open (downloadable) or closed (API-only) determines who can build on top of them.

🔓 Open Weights
Models like Llama, Mistral, Qwen release their weight files. Anyone can download, modify, fine-tune, or run them locally. This democratizes access but also means the model can be studied, red-teamed, or distilled by competitors.
🔒 Closed Weights
Models like GPT-4, Claude, Gemini keep weights secret. You interact via API only. The provider controls inference, pricing, and safety filters. The model itself is a black box — you can't inspect or modify it.

💡 Why It Matters
Open weights enable the entire downstream ecosystem: local LLMs, fine-tuned specialists, on-device inference, and distillation into smaller models. Closed weights fund frontier research through API revenue but create dependency on the provider's infrastructure.

Knowledge Distillation Concept

A teacher model (large, capable, expensive) trains a student model (small, fast, cheap) by sharing not just final answers, but the full probability distribution — the "soft targets" that encode nuanced reasoning.

🎓 How It Works
Instead of training on raw data labels, the student learns to mimic the teacher's output probabilities. If the teacher says "cat: 0.7, dog: 0.2, fox: 0.1", the student learns that subtle distinction — capturing far more information than a hard label of just "cat."

💡 Real-World Impact

Quantization Concept

Quantization reduces the precision of weight values. A 32-bit float becomes 16-bit, 8-bit, or even 4-bit. Fewer bits = smaller file, faster inference, less memory — but potentially lower accuracy.

📊 The Precision Trade-off
FP32 (32-bit): Full precision, huge files.
FP16/BF16 (16-bit): Half precision, nearly identical quality, 2x smaller.
INT8 (8-bit): 4x smaller, slight quality drop.
INT4 (4-bit): 8x smaller, often acceptable quality loss.
1-bit (BitNet): Extreme compression, emerging research.

💡 Why Quantization Shapes the App Layer
A 70B model at FP32 needs 280GB VRAM — impossible for consumer hardware. At INT4, it fits in 35GB. At INT8, a 7B model fits on a phone. Quantization is the bridge that makes on-device AI feasible.

Mixture of Experts New

MoE is a conditional computation architecture where only a subset of the model's parameters activate per token. A router learns which "expert" subnetworks to invoke for each input, enabling massive parameter counts without proportional compute cost.

🧩 The Core Idea
Traditional transformers use the same parameters for every token. MoE replaces dense feed-forward layers with sparse expert layers — multiple specialized networks where only top-k experts fire per token. DeepSeek-V3 (671B total, 37B active) and Mixtral 8x7B (8 experts, 2 active) are leading examples.

💡 Why MoE Changes the Economics
MoE lets models scale to trillions of parameters while keeping inference costs comparable to much smaller dense models. The catch: all experts must reside in memory, so VRAM requirements don't shrink — but compute per token does. This is why GPT-4, DeepSeek-V3, and Kimi K2 all use MoE.

Speculative Decoding New

Draft-then-verify: a small, fast "draft" model predicts several future tokens ahead. The large target model verifies them all in one parallel forward pass, accepting the matching prefix. This breaks the sequential token-by-token bottleneck, yielding 2-4x speedups with zero quality loss.

🚀 How It Works
Autoregressive generation is inherently sequential — each token needs a full forward pass. Speculative decoding pairs a lightweight draft model with the target model. The draft model rapidly proposes 3-5 tokens; the target verifies them in parallel. Accepted tokens are emitted; the first mismatch is corrected and the cycle repeats.

💡 The Zero-Quality-Loss Guarantee
Unlike quantization or pruning, speculative decoding is mathematically equivalent to running the target model alone. The draft model merely accelerates — it never changes the output distribution. This makes it the safest optimization technique for production deployments where quality cannot be compromised.

Agent Architecture New

An AI agent is not "an LLM in a loop." It's a complete production architecture with six layers: reasoning engine, memory, tools, planner, runtime, and observability. The agent perceives, reasons, acts, and observes in a continuous loop until the goal is achieved.

🤖 The Six Layers
1. LLM Core — The reasoning brain (Claude, GPT, Llama).
2. Memory — Short-term (context), episodic (vector DB), procedural (learned heuristics).
3. Tools — External actions via MCP, APIs, code execution, browser automation.
4. Planner — ReAct, Plan-and-Execute, Tree-of-Thoughts, Reflexion.
5. Runtime — LangGraph, CrewAI, OpenAI Agents SDK — manages state, retries, handoffs.
6. Observability — Langfuse, LangSmith — every step traced and evaluable.

💡 From Chatbot to Agent
A chatbot reacts to questions. An agent realizes goals. "What's the weather?" → chatbot. "Find me a cheap hotel with a pool, check reviews, compare 3 options, and book" → agent. The difference is autonomous planning and tool use across multiple reasoning steps.

The Application Layer Concept

Everything upstream — weights, distillation, quantization, MoE, speculative decoding, agents — converges here. These decisions determine what apps can exist, where they run, who can build them, and what they cost.

🌍 The Capability Cascade
Frontier Lab trains massive model → Open weights democratize access → Distillation creates efficient variants → MoE scales parameters without compute → Quantization shrinks for devices → SpecDec speeds inference → Agents add reasoning → Developers build previously impossible apps.

🧠 The Single Mental Model

Think of AI capability as water flowing downhill. Frontier labs are the mountain spring. Open weights unlock the reservoir gates. Distillation builds aqueducts to direct flow. MoE creates multiple channels so more water moves without widening the riverbed. Quantization filters and compresses the flow for the smallest villages. SpecDec adds pressure pumps for speed. Agents are the irrigation systems that decide where the water goes. The app layer is what grows in the valley — entirely dependent on whether the water reaches it.
